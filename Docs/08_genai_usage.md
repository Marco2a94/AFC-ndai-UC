# Generative AI usage

This repository documentation includes a section on GenAI usage because the tender requires explaining:
- **Where** GenAI was used
- **Why** it was used
- **How** it was used (examples)

## What we used GenAI for (examples)
- Drafting documentation structure and wording (runbook, architecture explanation)
- Reviewing code structure and suggesting improvements (readability, folder layout)
- Suggesting validation checks for ingestion (data types, missing values)
- Suggesting SQL view ideas for dashboards

## Verification
All GenAI-produced content was:
- Reviewed by the team
- Adapted to match the project requirements and the implemented code
- Tested by running the containers and verifying outputs

## What GenAI did NOT do
- No automatic deployment without review
- No uncontrolled copy/paste of generated code into production: every change was validated locally
